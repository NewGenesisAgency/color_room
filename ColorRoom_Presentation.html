Le besoin

Problématique et objectifs

Comment piloter et mesurer la lumière de 42 dalles LED, tout en proposant des jeux éducatifs créables facilement, sur un système autonome et hors ligne ?
  • Piloter les dalles en temps réel via le réseau (couleur, intensité)
  • Mesurer réellement la lumière émise (colorimètre Konica CS-160)
  • Offrir des jeux et un éditeur pour en créer sans coder
  • Fonctionner 100 % en local sur Raspberry Pi
  • Multijoueur : chaque joueur pilote une dalle depuis son téléphone

React / Next.js / Node / TS  vs  JS pur + Node-RED

Pourquoi cette stack ?

Approche initiale : Node-RED + JS pur
  • Node-RED : top pour prototyper l'IoT, limité pour une UI riche (3D, éditeur)
  • Flux en JSON difficiles à versionner et relire dans Git
  • JS pur : aucun typage, erreurs vues seulement à l'exécution
  • Refactorisation risquée, autocomplétion faible
  • Complexité vite ingérable sur un gros projet
Choix retenu : Next.js + React + Node + TS
  • React : interface en composants réutilisables
  • Next.js : front et back (API) dans un seul projet
  • Node.js : même langage côté serveur et navigateur
  • TypeScript : typage strict, bugs attrapés à la compilation
  • Du code texte : versionnable et testable (tsc + build)

Docker / Raspberry Pi

Réseau et déploiement

  • Orchestration via docker-compose.yml (4 services)
  • color-room : l'application (port 8080)
  • ollama + ollama-pull : IA locale (port 11434)
  • portainer : supervision des conteneurs (9000)
  • Réseau local : tablettes et téléphones sur le même Wi-Fi
  • Secrets isolés dans un fichier .env (non versionné)
  • Mise à jour : git pull puis docker compose up
# déploiement sur le Pi
git pull origin main
docker compose up -d --build

Une dalle par joueur

Multijoueur en réseau local

  • Chaque joueur ouvre la page sur son téléphone
  • Il scanne un QR code ou tape l'adresse locale
  • Il reçoit une dalle (un siège) et choisit sa couleur
  • Sa dalle s'allume en temps réel dans la Color Room
  • L'animateur peut jouer aussi depuis la tablette
  • Tout passe par le Wi-Fi local (pas d'Internet)

Bonnes pratiques

Sécurité

  • Mots de passe hachés (PBKDF2-SHA512, 100 000 itérations, sel) : jamais en clair
  • Sessions par cookie httpOnly + rôles (élève, enseignant, admin)
  • Secrets dans .env, exclu de Git via .gitignore
  • Transactions SQL atomiques (ACID) pour des états cohérents
  • Réseau local isolé : pas d'exposition Internet, IA hors ligne possible
  • Retour d'expérience : secret committé par erreur, purgé de l'historique Git

Difficultés rencontrées et solutions

! Lag de la vue 3D sur tablette
→
Textures allégées, pixel ratio plafonné, snapshots dédupliqués
! Contextes WebGL bloqués (crash)
→
Libération explicite du contexte + repli gracieux
! Son d'un jeu joué sans fin
→
Bug d'unité ms / secondes corrigé dans l'audio
! Tetris injouable sur 6x7
→
Refonte en jeu de combinaison de couleurs (pièces 1-2 cases)
